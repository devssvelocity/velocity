## 🐛 Common Issues and Fixes

### ❌ Velocity crashes Roblox on injection
If you are experiencing **velocity executor crashing roblox**, this is almost always caused by an outdated build conflicting with a recent Roblox client update.

**Fix:** Update Velocity to the latest version using the auto-updater or re-download from this repository. Ensure you are running the executor as Administrator and that no antivirus tool is interfering mid-injection.

---

### ❌ "Failed to inject" error on launch
**Fix:** Make sure Roblox is fully in-game (not on the loading screen or main menu) before launching Velocity. Restart both Roblox and Velocity, then try again.

---

### ❌ Antivirus quarantines the file
**Fix:** This is a false positive — all Roblox injectors trigger heuristic antivirus flags due to memory manipulation. Add the Velocity folder to your exclusion list *before* extracting the ZIP.

---

### ❌ Script Hub shows blank / fails to load
**Fix:** Check your internet connection. The Script Hub pulls from a live server. Temporarily disable your VPN or firewall and try again. If the issue persists, check the Discord server for any announced outages.

---

### ❌ macOS: "App is damaged" error
**Fix:** Open **Terminal** and run:
```bash
xattr -cr /Applications/Velocity.app
```
This removes the macOS quarantine flag applied to unsigned applications downloaded from the internet.

---

### ❌ Custom UI layout resets after update
**Fix:** Export your UI profile before updating via Settings → Export Layout. After updating, re-import it from the same menu. This is a known issue being addressed in v4.2.0.

---

### ❌ Script executes but produces no in-game effect
**Fix:** Confirm the script is designed for the specific game you are currently in. Many scripts are game-specific and will silently fail in the wrong environment. Filter the Script Hub by game name to find compatible scripts.
